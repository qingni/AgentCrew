# 设计文档：双模式选择器规则 + UI 文案草案（v0.1）

## 文档日期

2026-03-12

## 背景

AgentCrew 计划支持双模式并存：

- **Pipeline 模式**：固定步骤执行，速度快、成本低、行为可预测
- **Agent 模式**：轮次化执行，可根据中间结果动态重规划

为降低用户选择成本并避免误选，需要引入“模式选择器”：

1. 在任务开始前给出可解释的模式推荐
2. 在运行中根据风险和失败信号给出切换建议
3. 提供一致、清晰、可操作的 UI 文案

---

## 目标与范围

## 目标

1. 提供稳定、可解释、可覆盖的推荐机制（推荐而非强制）
2. 将建议触发点标准化（创建前 / 执行中 / 失败后）
3. 降低噪音提示，避免频繁打扰（防抖 + 冷却）
4. 在切换模式时保留已有进度，减少用户损耗

## 非目标（v0.1 不做）

1. 自动无感切换模式（不经过用户确认）
2. 多会话并发下的全局推荐优化
3. 基于复杂用户画像的个性化推荐策略

---

## 设计原则

1. **用户选择优先**：系统只推荐，最终由用户决策
2. **理由可见**：每次建议必须展示 2-3 条可读原因
3. **低打扰**：同原因不重复提示，设置会话冷却窗口
4. **可回退**：模式切换后可回退且不丢失执行产物
5. **可评估**：所有建议与用户决策均可埋点追踪

---

## 模式选择规则（v0.1）

### A. 任务创建前（Pre-run）评分推荐

采用 `0-100` 综合评分；分值越高越偏向 Agent 模式。

| 信号 | 触发条件 | 分值 |
|------|----------|------|
| 任务复杂度 | 阶段 >= 3 或步骤 >= 6 | +15 |
| 闭环需求 | 包含“实现 + 审查 + 修复/验证” | +20 |
| 不确定性 | prompt 含“探索/评估/可能/尝试”等 | +10 |
| 高风险域 | auth / payment / migration / security | +20 |
| 多工具协作 | 预计使用 >= 2 种工具 | +10 |
| 历史失败 | 同类任务最近失败率较高 | +15 |
| 用户偏好 | 用户长期偏好 Pipeline | -20 |
| 历史稳定 | 同类任务 Pipeline 连续成功 >= 3 次 | -15 |

#### 推荐阈值

- `>= 65`：强推荐 **Agent**
- `40 ~ 64`：弱推荐（双按钮并列，无主推）
- `< 40`：推荐 **Pipeline**

#### 硬触发（直接推荐 Agent）

满足任一条件即直接推荐 Agent：

1. 高风险域 + 预计包含 review/fix 闭环
2. 用户明确要求“自动重试 / 自动重规划 / 自我修复”

---

### B. 执行中（Runtime）切换建议

#### Pipeline -> Agent 建议触发

满足任一条件：

1. 连续失败步骤 `>= 2`
2. Reviewer 报告出现 `high/critical`
3. 同一步重试后仍失败
4. 长时间无进展（例如 15 分钟无有效产出）

#### Agent -> Pipeline 降级建议

满足全部条件：

1. 当前轮任务简单（<= 3 步，无跨角色依赖）
2. 连续 2 轮未触发 RePlan
3. 用户明确偏好“更快/更省”

---

### C. 防抖与降噪规则（Anti-flapping）

1. 同一会话建议弹窗最短间隔：`10 分钟`
2. 同一原因仅提示一次（除非严重级别升级）
3. 用户点击“本次不再提示”后，本会话内不再重复打扰
4. 单会话最多主动打断 `2` 次

---

## 模式切换行为定义

### Pipeline -> Agent

1. 从当前失败点创建 Agent Session
2. 已完成步骤标记为只读产物（artifacts）
3. 不覆盖原 Pipeline，保留回退入口

### Agent -> Pipeline

1. 将当前已确认计划导出为静态 Pipeline
2. 保留 Agent 会话记录用于审计与复盘
3. 用户可选择“立即切换执行”或“仅保存草稿”

## UI 文案草案（v0.1）

### 1) 创建任务页：模式卡片

#### Pipeline 卡片

- 标题：`Pipeline 模式`
- 副文案：`固定步骤执行，速度快、成本低、行为可预测`
- 标签：`适合：确定性任务`

#### Agent 卡片

- 标题：`Agent 模式`
- 副文案：`按执行结果动态调整，适合复杂任务闭环`
- 标签：`适合：实现 + 审查 + 修复`

#### 推荐提示条

- 主文案：`建议：Agent（任务复杂度较高，且需要审查修复闭环）`
- 次文案：`你仍可手动选择 Pipeline`

#### 按钮文案

- 主按钮：`使用 Agent（推荐）`
- 次按钮：`继续使用 Pipeline`

---

### 2) Pipeline 执行中：切换建议 Banner

- 标题：`建议切换到 Agent 模式`
- 正文：`检测到连续 2 个步骤失败，且存在高风险审查项。Agent 模式可自动重规划后续步骤。`
- 按钮：
  - `切换到 Agent（推荐）`
  - `继续当前模式`
  - `本次不再提示`

---

### 3) 切换确认弹窗

- 标题：`切换到 Agent 模式？`
- 正文：`将从当前失败点创建新会话，并保留已完成步骤结果。不会覆盖现有 Pipeline。`
- 信息项：
  - `已完成步骤：3`
  - `失败步骤：Review`
  - `将生成：1 个修复轮次（可人工确认）`
- 按钮：
  - 主：`确认切换`
  - 次：`取消`

---

### 4) 失败后总结页推荐

- 标题：`下次可尝试 Agent 模式`
- 正文：`本次失败主要发生在固定流程无法自适应调整的阶段。Agent 模式可自动补充“修复 + 验证”步骤。`
- 按钮：
  - `基于本任务创建 Agent 会话`
  - `保留 Pipeline 重新编辑`

---

### 5) 推荐理由展示模板（Explainability）

每次推荐显示 2-3 条理由，示例：

1. `任务包含高风险域：认证/安全`
2. `预期存在实现 -> 审查 -> 修复闭环`
3. `同类任务在 Pipeline 模式下失败率偏高`

## 默认参数建议（v0.1）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| preRunStrongRecommendThreshold | 65 | 创建前强推荐 Agent 阈值 |
| runtimeSwitchThreshold | 70 | 执行中建议切换阈值 |
| consecutiveFailureTrigger | 2 | 连续失败触发值 |
| agentMaxRoundsDefault | 4 | Agent 默认最大轮次（保证重试/局部修复/全局重规划链路可用） |
| stuckMinutesTrigger | 15 | 长时间无进展触发值（分钟） |
| recommendationCooldownMinutes | 10 | 建议冷却窗口（分钟） |
| maxInterruptionsPerSession | 2 | 单会话最大主动打断次数 |

> 说明：以上为工程默认值，需结合真实运行数据持续校准。

---

## 埋点与评估建议

为便于后续调参与评审，建议记录以下事件：

1. `mode_recommendation_shown`（展示推荐）
2. `mode_recommendation_accepted`（接受推荐）
3. `mode_recommendation_dismissed`（拒绝推荐）
4. `mode_switched_runtime`（运行中切换）
5. `task_outcome_by_mode`（按模式统计任务结果）

关键指标：

- 推荐采纳率（Acceptance Rate）
- 切换后成功率提升（Post-switch Success Lift）
- 平均完成时长（Lead Time）
- 每任务 token 成本（Cost per Task）
- 用户打扰率（Interruptions per Session）

---

## 上线建议

1. **阶段 1**：仅开启创建前推荐，不启用运行中打断
2. **阶段 2**：对失败率高的任务类型灰度开启运行中建议
3. **阶段 3**：根据埋点结果调整阈值与文案

原则：先确保“推荐可信”，再放大“自动建议力度”。
